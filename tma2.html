Belajar Microsoft Access: Pengenalan Dasar untuk Pemula



Apa itu Microsoft Access ?

Microsoft Access merupakan program database yang cukup populer, mudah digunakan, berorientasi visual dan berbasis Windows serta dapat diintegrasikan dengan aplikasi lain terutama Visual Basic.

Fungsi Umum Microsoft Access

  • Untuk membuat database
  • Untuk membuat program aplikasi berbasis desktop

Komponen Access

  1. Tables, adalah objek database yang digunakan sebagai sarana untuk menyimpan data yang telah diolah.
  2. Form, adalah objek database yang digunakan untuk memasukkan dan mengedit data atau informasi yang ada dalam suatu database dengan menggunakan bentuk tampilan formulir.
  3. Independensi Data: Data dapat diubah tanpa mempengaruhi aplikasi yang menggunakannya.
  4. Report, adalah objek database yang digunakan untuk menampilkan data atau informasi dalam bentuk laporan.
  5. Queries, adalah objek database yang berfungsi untuk menampilkan, menyunting dan menyaring suatu data sesuai dengan kriteria yang diinginkan serta dapat memasukkan suatu expresi.
  6. Macros, adalah rangkaian perintah yang dapat disimpan dan dijalankan secara otomatis, misalnya membuka form, mencetak laporan di layar, dan lain-lain.
  7. Moduls, adalah program kecil atau procedur yang kegunaannya adalah untuk perancangan modul aplikasi pengolahan database tingkat lanjut.

Tipe Data di Access

  1. Text: Tipe data ini digunakan untuk menyimpan teks atau huruf. Cocok untuk nama, alamat, atau informasi lain yang berbentuk kata-kata. Panjang maksimum untuk tipe data ini adalah 255 karakter.
  2. Memo: Jika kamu perlu menyimpan teks yang lebih panjang, seperti deskripsi atau catatan, gunakan tipe data ini. Memo dapat menampung hingga 65.535 karakter, jadi sangat berguna untuk informasi yang lebih rinci..
  3. Number: Tipe ini digunakan untuk menyimpan angka. Cocok untuk data yang memerlukan perhitungan, seperti jumlah, harga, atau nilai. Ada beberapa subtipe dalam Number, seperti Integer dan Double, yang berbeda dalam kapasitas dan jenis angka yang bisa disimpan.
  4. Date/Time: Seperti namanya, tipe data ini digunakan untuk menyimpan informasi tanggal dan waktu. Misalnya, kamu bisa mencatat tanggal lahir, tanggal transaksi, atau waktu kejadian.
  5. Currency: Tipe data ini khusus untuk menyimpan angka yang berkaitan dengan uang. Dengan menggunakan tipe ini, kamu bisa melakukan perhitungan dengan lebih tepat, karena Access menghindari kesalahan pembulatan yang sering terjadi pada tipe data angka biasa.
  6. AutoNumber: Ini adalah tipe data yang sangat berguna untuk membuat nomor unik secara otomatis setiap kali kamu menambahkan data baru. Biasanya digunakan sebagai ID untuk setiap record dalam tabel, sehingga tidak perlu repot-repot mengatur nomor manual.
  7. Yes/No: Tipe data ini sangat sederhana, hanya memiliki dua pilihan: "Ya" atau "Tidak". Cocok untuk pertanyaan yang hanya memerlukan jawaban biner, seperti apakah suatu produk tersedia atau tidak.
  8. OLE Object: Tipe data ini digunakan untuk menyimpan objek dari aplikasi lain, seperti gambar, dokumen, atau file audio. Misalnya, kamu bisa menyimpan foto produk langsung di database.
  9. Hyperlink: Jika kamu ingin menyimpan URL atau link ke situs web, tipe data ini adalah pilihan yang tepat. Dengan hyperlink, kamu bisa langsung mengakses halaman web hanya dengan mengklik link tersebut.
  10. Attachment: Tipe ini memungkinkan kamu untuk menyimpan berbagai jenis file dalam satu record, seperti dokumen, gambar, atau spreadsheet. Ini sangat berguna untuk mengelola dokumen yang terkait dengan data tertentu.
  11. Calculated: Tipe data ini memungkinkan kamu untuk menyimpan hasil perhitungan yang didasarkan pada data lain dalam tabel. Misalnya, kamu bisa menghitung total harga berdasarkan jumlah dan harga per unit.
  12. Lookup Wizard: Tipe data ini membantu kamu untuk membuat daftar pilihan dari tabel lain atau dari daftar yang sudah ada. Ini memudahkan pengguna dalam memilih nilai tanpa harus mengetiknya manual, sehingga mengurangi kesalahan input.

Primary Key dan Foreign Key

  • Primary Key

    Tabel memiliki primary key, yaitu suatu atribut yang tidak hanya mengidentifikasi secara unik suatu kejadian tetapi juga mewakili setiap kejadian dari suatu entitas. Contoh : NIM dalam tabel Mahasiswa merupakan nilai unik yang tidak mungkin bersifat ganda. Karena setiap mahasiswa memiliki NIM yang berbeda antara mahasiswa yang satu dengan mahasiswa yang lain.

  • Foreign Key

    Foreign key adalah atribut yang melengkapi relationship dan menunjukkan hubungan antara tabel induk dengan tabel anak. Foreign key ditempatkan pada tabel anak.

Jenis- Jenis Relasional

  1. One-To-One (1-1)

    Mempunyai pengertian "Setiap baris data pada tabel pertama dihubungkan hanya ke satu baris data pada tabel ke dua". Contohnya : relasi antara tabel mahasiswa dan tabel data pribadi. Satu baris mahasiswa hanya berhubungan dengan satu baris data pribadi begitu juga sebaliknya. Contoh relasi one-to-one



  2. One-To-Many (1-N)

    Mempunyai pengertian "Setiap baris data dari tabel pertama dapat dihubungkan ke satu baris atau lebih data pada tabel ke dua". Contohnya : relasi perwalian antara tabel dosen dan tabel mahasiswa. Satu baris dosen atau satu dosen bisa berhubungan dengan satu baris atau lebih mahasiswa.



  3. Many-To-Many (N-M)

    Mempunyai pengertian "Satu baris atau lebih data pada tabel pertama bisa dihubungkan ke satu atau lebih baris data pada tabel ke dua". Artinya ada banyak baris di tabel satu dan tabel dua yang saling berhubungan satu sama lain. Contohnya : relasi antar tabel mahasiswa dan tabel mata kuliah. Satu baris mahasiswa bisa berhubungan dengan banyak baris mata kuliah begitu juga sebaliknya.
